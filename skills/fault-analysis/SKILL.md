# 故障智能分析

对设备故障进行深度分析，不仅列出故障，还要分析根因、关联性和建议措施。

## 使用方法

1. 使用 `device_logs` 工具获取故障数据，按严重程度筛选
2. 使用 `device_status` 查看具体设备的运行状态
3. 综合故障频率、严重程度、设备运行时长进行分析
4. 给出分级建议：
   - 🔴 立即处理：critical 级别未解决的故障
   - 🟡 尽快安排：高频故障设备或同一类型反复出现的问题
   - 🟢 常规关注：低级别已解决的故障
5. 如果发现同类故障集中在某区域或某机型，要主动指出关联性
